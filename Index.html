    DNS lookup is the process of finding the IP address of a website name.
    For example: You type: google.com But computers communicate using IP addresses like: 142.250.190.14 So the computer asks DNS: “What is the IP address of google.com?” DNS replies with the correct IP address. This process is called a DNS lookup.
    📌 Full Working of DNS Lookup 1. You enter a website name Example: youtube.com 2. Browser checks cache Your browser first checks: Browser cache Operating system cache
    If the IP is already saved, it uses it directly. 3. Request goes to DNS Resolver If not found, the request goes to a DNS resolver (usually from your internet provider or services like Google DNS or Cloudflare). Example public DNS: 8.8.8.8 → Google DNS 1.1.1.1 → Cloudflare DNS 4. Resolver asks Root DNS Server The root server says:
    “I don’t know the exact IP, but ask the .com server.” 5. TLD Server responds TLD = Top Level Domain For youtube.com, the .com server says: “Ask YouTube’s authoritative DNS server.” 6. Authoritative DNS gives final IP The authoritative server returns the real IP address. Example:
    youtube.com → 142.xx.xx.xx 7. Browser connects to website Now the browser can contact the web server and load the website. 🔁 Simple Flow Website Name ↓ DNS Resolver ↓ Root Server ↓ TLD Server (.com) ↓ Authoritative DNS ↓ IP Address Found 📌 Why DNS is Important Without DNS:
